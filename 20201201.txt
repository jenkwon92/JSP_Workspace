웹어플리케이션 및 JavaEE
1
다음 설명 중 틀린 것은?

(1) Java 언어로 개발할 수 있는 분야는 크게 JavaSE, JavaEE, JavaME 의 3가지 플랫폼이 있다. o

(2) JavaSE로 제작된 독립 실행형 응용프로그램을 사용하려면,  해당 PC에 자바 실행 코드가 존재해야 
    한다. o

(3) 독립실행형 응용프로그램이 일단 배포가 된 후, 수정사항이 발생하였다면 코드는 재배포되어야 한다. o 

(4) 소프트웨어 개발회사가 (3)번과 같은 업무 방식으로 어플리케이션을 업데이트하는  경우, 많은 인력과
      비용을 줄일 수 있다. x

(5) IT가 발전되어질수록 배포 방법은 더욱 복잡해지고 있다. x


2
다음 설명 중 웹어플리케이션에 대한 설명 중 틀린 것은?

(1)독립실행형 응용프로그램이 사용자의 PC에 배포되어 직접 실행되어지는 반면, 웹어플리케이션은 실행
    코드를 클라이언트가 보유하지 않고, 서버측에 집중시킨다. o

(2)클라이언트는 코드를 직접 수행하지 않으므로, pc사양이 좋을 필요가 없으며 단지, 서버측에서 실행된
    결과를 해석할 전용 프로그램이 수행될 수 있는 성능 수준이면 된다  o

(3)서버 중 웹서버는 자신이 수행한 결과를 클라이언트와 주고받을때, 전자문서를 주고 받는 프로토콜로
    통신을 수행하는데 이러한 프로토콜을 특히 HTTP 프로토콜이라 한다.  o

(4) 결국 웹어플리케이션은 중앙의 서버가 실행을 담당하고, 이 결과를 클라이언트의 브라우에서 제공받는
     방식으로 실행된다. o
 
(5)유지보수성이나 배포방법을 고려했을때 기업용 어플리케이션은 당연히 독립실행형 어플리케이션 우위에
    있다.  o

3
다음 설명 중 웹어플리케이션에 대한 설명 중 틀린 것은?

(1)하나의 웹어플케이션이 개발 및 구축되려면 서버측 수행을 담당하는 서버개발자, 서버가 실행한 결과를
    받아와 웹표준을 준수하여 화면에 보여줄 웹퍼블리셔, 그 화면을 구성하는 코드가 단순히 디자인 국한된 
    것이 아닌 동적 기능(프로그래밍)이 구현되어야 할 경우엔 프론트개발자, 이렇게 서버와 클라이언트
   분야 마다 전문가들의 협업이 필요하다 o

(2)웹어플리케이션은 데스크탑에서 실행되는 독립 실행형 어플리케이션에 비해 다룰 수 있는 프로그램
    기술적 영역이 더 넓다  x

(3)응용어플리케이션보다 배포가 용이하고 유지 보수성이 높다. o 

(4)자바의 3가지 개발 플랫폼 중 웹어플리케이션 구축에 필요한 기술 분야는 JavaEE이다. o


4
다음 중 맞는 것을 고르면?

(1)Java로 독립실행형 어플리케이션을 개발하려면 JDK를 설치해야 하며, 이 JDK에는 응용프로그램개발에
    필요한 각종 API가 포함되어 있다. o

(2)Java가 서버에서 실행되려면, 기존 javaSE와는 다른 방식으로 수행될 API,  즉 서버전용 API 가 필요하다 o

(3)(2)번과 같이 서버에서 실행되는 API를 사용하려면 개발하려면 JEEDK를 설치해야 한다. x <존재하지않음>

(4)javaSE와는 달리 javaEE개발시 개발자가 별도로 설치할 전용 SDK는 없다. o
 
(5)JavaEE기반의 어플리케이션에 사용되는 API는 Sun에서 규정한 javaEE스팩을 준수하는 웹서버가 보유
   하고 있으므로, 개발자는 웹어플리케이셔 개발을 위해서 단지 웹서버만 설치하면 된다. o

	javaEE 서버 기반의 종류(유,무료)
-------------------웹 컨테이너---------------------
		Tomcat (아파치 재단) -무료
		Resin - 유료
-------------------웹 어플리케이션 서버(WAS)--------
	웹 컨테이너 + EJB컨테이너
	지금은 사용하지 않지만, EJB(Enterprise Java Bean) 실행시킬 수 있다.
		WebLogic (Oracle) - 유료 : 강력하므로 전세계적으로 가장 유명함
		Jeus(티맥스)한국 - 유료 : 인지도가 낮아서 가격대가 낮음
		JBoss(RedHat) - 유료(기술 서비스 제공),무료(도와주지 않음)

다음 중 틀린 것을 고르면?

(1)Java 언어로 오라클,MySQL 등을 제어하려고 할때  DBMS제품에 따라 연동소스코드에 큰 변화가 있다. x

(2)사용하려는 DBMS마다 필요한 드라이버 jar는 Sun에서 배포한다. x

(3)DBMS제어에 사용되는 드라이버마다 접속객체, 쿼리수행객체, 결과집합 객체의 명칭이 각각 다르다 x

(4)접속객체, 쿼리수행객체, 결과집합 객체의 명칭이 모든 드라이버마다 동일한 이유는 Sun에서는 소스
   코드를 직접 작성한게 아니라, 단지 스팩만 명시하였고, 그 구현은 DBMS벤터사에게 위임하였기 때문이다 o  

(5)DBMS벤더사는 전세계 Java 개발자들에게 제품을 판매하려면 가급적 Sun에서 명시한 스팩을 준수하지
   않고, 창의적인 API명칭을 써야 할 것이다. x

다음 설명 중 틀린 것을 모두 고르면?

(1) javaEE 기반의 어플리케이션을 개발하기 위해서는 별도의 SDK 등을 설치 할 필요가 없다. o

(2) JavaEE 기반의 웹서비스를 구축하려면 JavaEE 기반의 코드가 실행될 수 있는 전용 서버 프로그램을
    이용해야 하는데, 이 서버 프로그램을 개발하는 업체는 바로  Sun 사이다 x sun에서는 개발에 참여하지않음
     
(3) JavaEE 전용 서버의 제조사마다 작동 방식이 틀리므로 , 제품이 바뀌면 자바 개발자들은 웹프로그램
    소스 코드도 변경해야 한다. x

(4) 자바 개발자들이 웹서버의 종류와 상관 없이 소스코드를 동일하게 유지할 수 있는 이유는, 자바 언어
     를 제작한 Sun에서는 javaEE 기반의 기술적 스팩만을 명시하고, 웹서버를 제작하는 밴더사들은 이
     스팩 및 기준에 따라 웹서버를 제작하였기 때문이다. o

(3) OS는 Window, DB는 Oracle , 웹서버는 Tomcat으로 구성한다.
(4) OS는 Unix, DB는 오라클 , 웹서버는 WAS로 구성한다.

다음 설명 중 틀린 것은?

(1) JavaEE 기반의 웹서버의 종류는 여러 가지가 있으며 오라클사의 WebLogic,레드햇의 JBoss, 티맥소
     프트의 JEUS 등과 같은  WAS와 , Apache의 Tomcat, caucho의 Resin과 같은  컨테이너가 있다

(2)웹서버를 Tomcat으로 개발하던 WebLogic으로 개발하던 클래스나 jar파일은 반드시 WEB-INF/lib 디렉
     토리에 두어야 한다는 스팩에는 변함이 없기 때문에 자바 기반의 웹어플리케이션은 어떠한 종류의
     웹서버에서건 코드나 환경설정에 영향을 받지 않고 개발이 가능하다.

(3)WebLogic, JBoss 등 javaEE기반의 WAS(웹어플리케이션서버)는 모두 Sun의 javaEE 스팩을 준수하여
   제작되었으므로, 서버 제작사가 각각 서로 기술력은 틀리지만 서블릿 클래스의 명칭이나 또는 그 보유
   메서드들에 대한 API는 동일하다. 따라서 javaEE 개발자 입장에서는 웹어플리케이션 서버가 바뀌었다
   고 해서 소스 코드가 변경되는 일은 없다.

(4)서블릿을 포함한 java EE 구현 객체를 재정의한 구현체인 jar파일은 Sun의 웹사이트를 통해 다운받으
     면 된다.
